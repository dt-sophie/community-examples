# Dynatrace GitHub Custom Agent

## Overview
**From code to cloud: Consolidating observability for developers.**

The **Dynatrace GitHub Custom Agent** introduces a powerful new entry point for Dynatrace insights directly within GitHub repositories. This integration ensures developers can access critical observability and security data without leaving their familiar GitHub environment. Whether it's triaging production errors, validating deployments, or responding to security vulnerabilities, the custom agent provides end-to-end visibility from code to production.

With the Dynatrace GitHub Custom Agent, you can:
- Investigate and resolve production failures directly within GitHub.
- Validate deployments by comparing pre- and post-deployment metrics.
- Monitor and categorize production errors to prioritize remediation efforts.
- Detect performance regressions before they impact end-users.
- Automate release validation and health checks.
- Respond to security vulnerabilities and ensure compliance monitoring.

## ⚙️ Getting Started

The **Dynatrace GitHub Custom Agent** helps organizations:
- Achieve true end-to-end visibility from code to production.
- Enhance developer productivity by integrating observability directly into GitHub workflows.
- Improve operational excellence by enabling faster incident response and root cause analysis.
- Ensure application reliability and security through automated monitoring and validation.

The custom agent bridges the gap between development and production by combining the capabilities of Dynatrace with GitHub Copilot and its custom agents. This integration empowers developers to act on insights immediately, reducing delays and inefficiencies in the software development lifecycle.
